# Case-study diagram generators

Every SVG under `src/assets/` is generated by one of these scripts. Edit the script
and regenerate — don't hand-edit the SVG output.

| Script | Output | What it draws |
|---|---|---|
| `genflows.mjs` | `src/assets/flows/` | The five Blend Metrics user flows + the legend |
| `genwf.mjs` | `src/assets/wireframes/` | Wireframe Iterations — full screens, V1 (low-fi) → V3 (mid-fi) |
| `gencomp.mjs` | `src/assets/wireframes/` | Wireframe Iterations — component panels and popup states |
| `genfinal.mjs` | `src/assets/final/` | Final Features — high-fidelity grayscale screens |

Each takes the output directory as its only argument. Use absolute paths:

```sh
node scripts/wireframes/genflows.mjs "$PWD/src/assets/flows"
node scripts/wireframes/genwf.mjs    "$PWD/src/assets/wireframes"
node scripts/wireframes/gencomp.mjs  "$PWD/src/assets/wireframes"
node scripts/wireframes/genfinal.mjs "$PWD/src/assets/final"
```

The SVGs are inlined into the page (`?raw` + `set:html`) rather than referenced via
`<img>`, so they inherit the page's DM Sans webfont. An `<img>` would fall back to a
system font and overflow the drawn boxes.

Note: `.cs-dot--*` annotation positions in `public/case-study.css` are percentages of
image height. If a `genfinal.mjs` screen changes height, its dots need re-measuring
against the annotation cards.
