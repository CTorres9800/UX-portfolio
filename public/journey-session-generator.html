Before Building

Creating a Workflow

  • Understand what the workflow builder does and how it connects to my tools
  • Name the workflow and land on a canvas that guides me to the first step
  • Find available triggers quickly without searching through menus
  • It's so hard to determine if this is what I'm looking for, if only I could change just one thing
  • The canvas is blank with no guidance. I don't know if I should drag or click
  • I didn't know there were other triggers available to me
  • Guided empty state with "Add a Trigger" prompt
  • Auto-open sidebar with searchable trigger list
  • Dotted drop zones visible by default
  • Floating "+" button for adding steps inline
  • Live Chat to ask questions to support
  • The canvas is too intimidating. I don't know if I should drag or click or what
  • Where are the other tools available?
  • Organize the builder by sections to allow for easier navigation and exploration
  • Include an explanation of the process to immediately inform newcomers what to expect
  • I don't have the funds to fix issues that happen from misconfiguration. I just need to get it right the first time.
Overwhelmed by the blank canvas, not sure where to start or what the first step should be. I feel like I need someone to hold my hand through the first setup.

Configuring Split Paths

Allow users to add conditional or percentage-based branching logic.

  • Did I make sure to choose the right split type? I want to route by conditions, not percentages
  • AND/OR conditions should be readable at a glance
  • Confirmation before deleting a path with steps inside
  • No explanation of when to use conditional vs. percentage split
  • Complex AND/OR logic is hard to read at a glance
  • Deleting a path with steps has no confirmation
  • Clear descriptions under each split type
  • Inline condition summaries on canvas paths
  • Merge paths toggle for downstream control
  • Confirmation dialog before deleting paths
  • If I edit, do I need to start at the beginning and navigate to the path I need to change? That might take a lot of scrolling
  • I need to be confident that the split logic sends the right contacts down the right path
  • It better have a double check. Editing and preview should be seamless, so the same context is preserved
  • I really hope I don't have to start all over
Worried I'll pick the wrong split type and have to redo the whole branch from scratch. The logic needs to be readable at a glance.
cautious
Condition summaries inline on canvas branches. Emphasized "Add Condition" buttons instead of "remove" formatting.

Testing

Allow users to validate the entire workflow before publishing it live.

  • This process is critical. I really want to see if the connections actually work
  • Is it clear? I'm not going to get surprised after publishing
  • Did I set all my fields and information correctly? I should double-check
  • No way to test before publishing
  • Error messages are vague and don't point to specific nodes
  • No visual indication of which connections failed
  • Pre-publish test runner with pass/fail per connection
  • Red error borders on misconfigured nodes
  • "Test Again" button for iterative debugging
  • Expandable details per failed connection
  • No Blockers
  • I'd like to make sure I understand the test results before publishing since this is a LIVE workflow
If this test fails I need to know exactly what broke so I can fix it without guessing. I can't afford to publish something broken.
anxious
Inline test results with pass/fail counts and expandable details per failed connection.

After Testing

Publishing

Allow users to publish their workflow live after testing passes.

  • I don't have time to second-guess. I need to know the test passed and then publish quickly
  • There needs to be some kind of confirmation so I know this is going live
  • I want options after publishing, not a dead end
  • No confirmation step before going live
  • No success state or celebration after publishing
  • No clear next-step options. Just a blank canvas
  • Two-step publish confirmation dialog
  • "This workflow will go live" in plain language
  • Success state with green checkmark
  • "Go To Dashboard" or "Keep Working" options
  • Do I have to re-test after every edit? If so, how quickly?
  • Can I send invites to other people to review the workflow?
  • Why were these test results not shared with stakeholders?
  • This part of getting my workflow live should feel like make or break. I want a graceful launch, not a silent button click
  • Seeing the success state helps confirm this is a real milestone
This is the point of no return. I need to be absolutely sure everything is right before going live. One wrong setting and it affects real people.
hesitant
Two-step publish confirmation with plain-language summary. Success celebration with dashboard and continue options.

Verify workflow is running correctly and manage existing automations.

  • How will I know when a workflow stops or breaks after it's live? Is there a way to track it?

  • No search or filter on the workflow dashboard
  • No version history to roll back changes
  • Deleting a workflow has no recovery option
  • Can't easily find old workflows by name
  • A screen where I can track all workflows, see status, and manage them
  • Search, filter, and sort by name, status, or date
  • Context menus for rename, duplicate, delete
  • Version history with rollback
  • I want out of those workflows to know which ones are SUPPOSED to be available. Why is no one actually available? So annoying!
  • It should take into account last activity, confirmed availability, and recent changes
  • I don't want to keep checking to see if someone has accepted my invite. I have other things to do!
I just want to see at a glance which workflows are running and which ones need attention. Don't make me dig through menus.
